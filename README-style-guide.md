# Руководство по стилю кода BAZAAR

## Общие правила

1. **Именование файлов**
   - Компоненты: `PascalCase.tsx`
   - Утилиты и хуки: `camelCase.ts`
   - Стили: `kebab-case.css`

2. **Импорты**
   - Использовать именованные импорты вместо пространства имен: `import { useState } from 'react'`
   - Импорт React не требуется при использовании JSX (настройка `"jsx": "react-jsx"` в tsconfig.json)
   - Импортировать только то, что используется
   - Порядок импортов:
     1. Внешние библиотеки
     2. Типы, модели, интерфейсы, утилиты
     3. Компоненты
     4. Стили

3. **TypeScript**
   - Всегда указывать типы для пропсов компонентов
   - Использовать интерфейсы для определения типов пропсов
   - Избегать `any`
   - Использовать точечные импорты типов: `import type { MouseEvent } from 'react'`

## Компоненты

1. **Структура компонентов**
   - Использовать функциональные компоненты вместо классовых
   - Объявлять компоненты как стрелочные функции без указания `React.FC`:
     ```tsx
     const Component = ({ prop1, prop2 }: ComponentProps) => {
       return (...);
     };
     ```

2. **Хуки**
   - Хуки вызывать в начале компонента
   - Использовать кастомные хуки для общего функционала
   - Для `useEffect` указывать все зависимости

3. **Пропсы**
   - Использовать деструктуризацию пропсов
   - Определять значения по умолчанию через параметры функции
   - Документировать назначение пропсов в интерфейсе
   ```tsx
   interface ButtonProps {
     /** Текст кнопки */
     text: string;
     /** Обработчик нажатия на кнопку */
     onClick?: () => void;
     /** Размер кнопки (s, m, l) */
     size?: 'small' | 'medium' | 'large';
   }
   ```

## CSS и верстка

1. **Методология БЭМ**
   - Блок: `block`
   - Элемент: `block__element`
   - Модификатор: `block--modifier` или `block__element--modifier`

2. **Правила верстки**
   - Семантическая верстка
   - Логический подход к отступам
   - Для позиционирования использовать Flexbox и Grid
   - Избегать inline стилей

3. **Доступность**
   - Использовать атрибуты `aria-*` для улучшения доступности
   - Добавлять `alt` к изображениям
   - Использовать семантические HTML-теги (`nav`, `article`, `header`, `footer` и т.д.)

## Производительность и оптимизация

1. **Мемоизация**
   - Использовать `useMemo` для сложных вычислений
   - Использовать `useCallback` для функций, передаваемых дочерним компонентам
   - Применять `React.memo()` для компонентов, получающих одинаковые пропсы

2. **Ленивая загрузка**
   - Использовать динамический импорт и `lazy` для компонентов страниц
   - Оптимизировать загрузку изображений

3. **Избегать избыточных рендеров**
   - Не создавать функции внутри JSX
   - Избегать использования стрелочных функций прямо в JSX
   - Вынести обработчики событий в отдельные переменные
   ```tsx
   // Плохо
   <button onClick={() => handleClick(id)}>Нажми меня</button>
   
   // Хорошо
   const handleButtonClick = useCallback(() => handleClick(id), [id]);
   <button onClick={handleButtonClick}>Нажми меня</button>
   ```

## Стандарты кода

1. **Форматирование**
   - Использовать eslint для проверки качества кода
   - Использовать prettier для форматирования
   - Отступы: 2 пробела
   - Максимальная длина строки: 100 символов

2. **Комментарии**
   - Использовать JSDoc для документирования функций и компонентов
   - Добавлять комментарии к сложным частям кода
   - Писать комментарии на русском языке

# Руководство по стилю Bazaar

## Цветовая схема

### Светлая тема
- **Основной цвет**: Теплое золото (#D4A977) с вариациями
- **Вторичный цвет**: Кремовый пастельный (#E8D6C6)
- **Акцентный цвет**: Приглушенное золото (#B89F7A)
- **Фон**: Очень светлый кремовый (#FFFBF6)
- **Текст**: Глубокий теплый серый (#4A433B)

### Темная тема
- **Основной цвет**: Золото (#D4A977) на черном фоне (#121212)
- **Акцентный цвет**: Приглушенное золото (#B89F7A)
- **Фон**: Глубокий черный (#121212)
- **Текст**: Кремовый белый (#F4E8DD)

## Типографика

- **Основной шрифт**: Montserrat (sans-serif)
- **Заголовки**: Cormorant Garamond (serif)
- **Акцентный шрифт**: Playfair Display (serif)

## Элементы дизайна

### Кнопки
- **Основные**: `button--primary` - Основные кнопки для ключевых действий
- **Контурные**: `button--outline` - Для менее приоритетных действий
- **Текстовые**: `button--text` - Для ссылок и несущественных действий

### Карточки
- Слегка приподнятые с тенью
- Эффект при наведении: приподнимаются выше, увеличивается тень
- Минималистичный дизайн с высоким контрастом между текстом и фоном

### Декоративные элементы
- **Разделители**: `.divider-accent` - Тонкие акцентные разделители
- **Рамки**: Минималистичные рамки с небольшим радиусом скругления
- **Тени**: Мягкие тени для создания глубины
- **Икония**: SVG иконки вместо эмодзи для улучшения доступности и масштабируемости

## Компонент Icon

Для всех иконок используйте компонент `Icon` вместо эмодзи или встроенных SVG:

```tsx
import Icon from '../common/Icon';

<Icon name="heart" size={18} color="currentColor" />
```

Доступные иконки:
- Социальные сети: `telegram`, `whatsapp`, `instagram`
- Системные: `sun`, `moon`, `cart`, `profile`
- Контакты: `location`, `clock`, `phone`, `check`
- Доставка: `truck`, `store`, `lightning`
- Ценности: `leaf`, `diamond`, `heart`, `handshake`

При необходимости добавления новых иконок, расширяйте интерфейс `IconProps` в `Icon.tsx`.

## Классы для использования

### Заголовки
- `.section-title` - Основной стиль заголовков секций
- `.section-title--centered` - Центрированный заголовок секции

### Декоративные элементы
- `.divider-accent` - Акцентный разделитель
- `.framed-image` - Изображение с рамкой

### Кнопки
- `.button` - Базовая кнопка
- `.button--primary` - Основная кнопка
- `.button--outline` - Контурная кнопка

## Структура страниц

Для страниц используйте следующую структуру:

```tsx
<div className="page-name">
  <div className="container">
    <h1 className="section-title section-title--centered">Заголовок страницы</h1>
    
    <section className="page-name__section">
      <h2 className="section-title">Заголовок раздела</h2>
      <div className="page-name__content">
        {/* Содержимое секции */}
      </div>
    </section>
    
    <div className="divider-accent"></div>
    
    {/* Дополнительные секции */}
  </div>
</div>
```

## Общая эстетика

Обновленный дизайн основан на принципах минимализма с акцентами:
- Четкие и читаемые шрифты
- Высокий контраст между текстом и фоном
- Четкие SVG иконки вместо эмодзи
- Минималистичные акценты в виде разделителей и иконок
- Адаптивность и доступность интерфейса

## Миграция со старого стиля

При обновлении существующих компонентов:
1. Заменяйте эмодзи на компонент `Icon`
2. Заменяйте классы `.fashion-heading` на `.section-title`
3. Заменяйте `.gold-frame` на обычные контейнеры с `.section-name__section`
4. Используйте `.divider-accent` вместо `.divider-gold`

## Правила верстки

- Использовать методологию БЭМ для классов
- Применять семантическую верстку
- Следовать логическому подходу к отступам
- Сохранять существующую сетку и отступы
